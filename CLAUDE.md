# Описание архитектуры и стандартов проекта

Этот файл содержит строгие правила по стеку технологий, структуре директорий и написанию кода. Всегда следуй этим инструкциям при генерации, рефакторинге или анализе кода.

## 1. Стек технологий

- **Frontend:** React, TypeScript, Vite.
- **Стилизация:** SCSS.

## 2. Глобальная структура репозитория

Проект представляет собой монорепозиторий. В корневой папке всегда создаются две основные директории:

- `client/` — frontend-часть (React/Vite).
- `server/` — backend-часть (Node.js/Express или др.).

Все команды, касающиеся фронтенда, должны выполняться внутри папки `client/`.

## 3. Структура папки Client

Фронтенд-приложение имеет следующую архитектуру внутри `client/src/`:

- `api/` — запросы к серверу.
- `assets/` — статические файлы (картинки, иконки, шрифты).
- `components/` — переиспользуемые UI-компоненты.
- `context/` — React Context.
- `hooks/` — кастомные React-хуки.
- `layouts/` — компоненты-обертки для страниц.
- `pages/` — компоненты целых страниц.
- `styles/` — глобальные стили и переменные (добавляются вручную).

В корне `client/src/` находятся точки входа: `App.tsx` и `main.tsx`.

## 4. Правила создания компонентов, страниц и лейаутов

Каждый компонент (а также страница или лейаут) должен располагаться в собственной папке.
Обязательный состав папки компонента `ComponentName`:

1. `ComponentName.tsx` — логика и разметка компонента.
2. `ComponentName.module.scss` (или `.scss`) — стили компонента.
3. `index.ts` — файл для удобного импорта с реэкспортом следующего вида:
   export { default } from "./ComponentName";

_Пример импорта такого компонента:_ `import ComponentName from '@/components/ComponentName';`

## 5. Настройка импортов (Path Aliases)

В проекте используются алиасы путей для чистоты импортов. Знак `@` всегда ссылается на директорию `client/src/`.
При генерации конфигураций (`vite.config.ts` и `tsconfig.json`) обязательно добавляй настройку алиасов:

- В `tsconfig.json`: `"paths": { "@/*": ["./src/*"] }`
- В `vite.config.ts`: настройка `resolve.alias` для `@`.
  Всегда используй абсолютные импорты через `@` вместо длинных относительных путей (напр., `../../`).

## 6. Базовая стилизация

Глобальная папка `styles` закидывается в проект вручную.
Для правильной работы стилей и сброса дефолтных настроек браузера обязательно подключение библиотеки. При инициализации проекта (или при обновлении зависимостей) учитывай, что должна быть установлена и импортирована следующая библиотека:
`npm install @a1rth/css-normalize@latest`
